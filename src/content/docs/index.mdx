## Ключевые концепции

<CardGrid>
	<Card title="Модель вместо хранилища" icon="database">
		Нет хранилища состояния - нет проблем. Состояние инкапсулировано в модели и реактивно. Вне модели доступно только для чтения и наблюдения. Уничтожение модели равно уничтожению состояния.
	</Card>
	<Card title="Действие - это объект" icon="play">
		Хранит операцию для изменения состояния. Имеет методы управления выполнением и свойства для контроля состояния выполнения. Определяется как обычный метод.
	</Card>
  <Card title="Контейнер моделей и зависимостей" icon="code">
		Не нужно думать как создать, получить или удалить модель после использования - за это отвечает контейнер.
	</Card>
	<Card title="Зависимости через внедрение" icon="code">
		Модель принимает зависимости как аргументы конструктора. Нет прямых импортов из других модулей. 
	</Card>
</CardGrid>

## Ключевые особенности

<CardGrid>
	<Card title="В основе ООП" icon="code">
		Модель определяется как стандартный класс. Наследование, инкапсуляция, полиморфизм, деструктор доступны по умолчанию.
	</Card>
	<Card title="Сохраняет типобезопасность" icon="check">
		Все подсказки автодополнения будут работать как внутри, так и вне контекста класса.
	</Card>
	<Card title="Реактивность Vue" icon="database">
		Всё работает на основе реактивности Vue. Не нужно изучать дополнительные API или подходы для создания реактивных объектов.
	</Card>
	<Card title="Простота тестирования" icon="check">
		`@vue-modeler/model` позволяет писать меньше кода, это значит нужно меньше тестов. Тестирование моделей не отличается от тестирования экземпляров обычных классов.
	</Card>
</CardGrid>
